분산처리의 예, 로드밸런싱(Load Balancing)

1.네트워크 트래픽 분산처리
=> 동시 접속자가 많을 경우 각 접속자에게 일정비율로 네트워크 트래픽 사용
=> 스위칭 허브 등의 장비 필요

2.서버를 여러개 병렬구축
=>접속자를 여러 서버로 고르게 배분

3.db를 여러개의 파일로 병렬구축
=>물리적으로 여러파일에 고르게 배분
=> 대량의 데이터처리에 효과적

4.Table을 파티션화. 파티션테이블
=>테이블단위로 데이터 분산처리

5.Server와 클라이언트 작업을 효과적으로 분산
=> 서버에서 하는 복잡한 로직(부하)가 있다면 클라이언트로 이관
=> 클라이언트에서 하던 일을 서버로 이관.

암호화 <-> 복호화

암호화 : 원본데이터를 식별불가한 값으로 변환
복호화 : 암호화된 데이터를 다시 원본데이터로 변환

쌍방향 암호화 : 암호화, 복호화 모두 가능
단방향 암호화 : 암호화 가능, 복호화 불가

비밀번호 암호화는 단방향

쌍방향 암호화는 일반적으로 key가 필요함.
