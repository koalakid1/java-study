기본키(Primary Key)
: 테이블에서 식별자 역할을 하는 컬럼
=> 중복을 방지해 줌
=> NOT NULL(값이 꼭 입력되어야함.)

ex) 회원의 테이블의 id
     상품테이블의 상품번호
     학생테이블의 학번

RDB에서는 모든테이블에 PK를 만들것을 권장

=================================

무결성 : 결함이 없음

1.not null : 데이터가 반드시 입력되도록 제약

2.primary key : 데이터가 중복되지 않도록 제약(Constraint)

3.foreign key : 참조 무결성을 유지. 참조하는 컬럼은 pk이어야 함.
